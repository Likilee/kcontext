# MVP 화면 및 기능 기획서 (Core UX Specification)

**Document Status:** Approved (MVP Core UX) **Product Name:** kcontext

본 문서는 로그인, 북마크 저장, 결제 검증(Fake Door), 제휴 마케팅 등의 부가적인 비즈니스 로직을 완전히 걷어내고, 서비스의 가장 본질적인 핵심 가치인 **초고속 문맥 검색과 완벽한 자막 동기화 시청(Search & Sync)** 경험에만 100% 집중한 MVP(Minimum Viable Product) 화면 및 기능 명세서입니다.

모든 화면은 모바일 퍼스트(Mobile-First) 기준으로 설계되며, 디자인 시스템의 다크 테마(bg-base)를 기본 캔버스로 사용합니다.

## 1. 공통 레이아웃 (Global UI)

앱 전반에 걸쳐 사용자의 끊김 없는 탐색을 지원하는 네비게이션입니다. MVP 단계에서는 사용자 상태(로그인/프로필) 관련 요소가 완전히 배제됩니다.

### 1.1. 상단 네비게이션 바 (GNB)

- 위치 및 배경: 화면 최상단 고정(Sticky). 스크롤 시 bg-base 기반에 블러(Blur) 효과를 적용하여 시각적 깊이감 제공. 좌우 여백은 space-layout-screen 강제 적용.
- 로고 (좌측): kcontext 텍스트 로고(text-primary, type-heading). 터치 시 홈 화면으로 라우팅되며 기존 검색 상태를 초기화.
- 글로벌 검색창 (우측/중앙 확장 - 핵심 연속 탐색 UI):
    - 홈 화면에서는 숨김 처리되며, '검색 결과 및 플레이어 화면' 진입 시 GNB 영역에 노출.
    - 영상을 시청하다가 다른 단어나 문법이 궁금해졌을 때, 뒤로 가기나 홈으로 돌아갈 필요 없이 즉시 새로운 검색을 실행할 수 있도록 지원.
    - 스타일: 높이가 얇은 캡슐 형태(radius-pill), bg-surface 배경, 돋보기 아이콘 포함.
    - 인터랙션: 검색어 입력 후 Enter 또는 돋보기 터치 시, 화면 전환(새로고침) 없이 즉각적으로 새로운 검색 결과 리스트를 페칭(Fetching)하여 플레이어 화면 갱신.

## 2. 홈 화면 (Home Screen)

서비스에 처음 진입한 사용자가 무엇을 검색해야 할지 모르는 상태(Blank Canvas Syndrome)를 방지하고, 오직 '첫 검색'이라는 단일 액션에 완벽히 집중시키는 랜딩 페이지입니다.

### 2.1. 히어로 검색 영역 (Hero Section)

- 메인 카피: "교과서에 없는 진짜 한국어, 원어민의 맥락으로 확인하세요." (text-primary, type-hero, 중앙 정렬). 상하 여백 space-layout-section 확보.
- 대형 검색창:
    - 형태: 화면 정중앙 배치. bg-surface 및 radius-pill 적용. 내부 패딩은 space-inset-squish를 스케일업하여 모바일 터치 영역 극대화.
    - 동적 플레이스홀더: 3초 주기로 검색 예시(예: "진짜 행복해요", "어쩔티비", "-잖아요") 타이핑 롤링 애니메이션 적용 (text-disabled, type-body).
    - 인터랙션: 터치 시 포커스 상태(border-focus) 활성화. 검색어 입력 후 Enter 또는 돋보기 아이콘 터치 시 검색 결과 화면으로 즉시 라우팅.

### 2.2. 추천 검색어 칩 (Suggestion Chips)

- 레이아웃: 대형 검색창 바로 하단에 배치. 모바일에서 가로 스와이프 가능한 리스트 형태. 요소 간 간격 space-gap-item 적용.
- 스타일: 캡슐 형태(radius-pill), 배경 bg-surface, 테두리 border-default, 텍스트 text-secondary.
- 콘텐츠 구성: 비즈니스 이메일 패턴, K-Drama 단골 리액션, 유행어 등 즉각적인 클릭을 유도하는 상황별 키워드 큐레이션 제공.
- 인터랙션: 칩 터치 시 motion-duration-fast (100ms) 축소 피드백 제공. 즉시 해당 키워드가 검색창에 입력되며 검색 실행.

## 3. 검색 결과 및 플레이어 화면 (Search & Player Screen)

유저가 가장 많은 시간을 보내는 핵심 화면입니다. PGroonga 검색 인덱스(DB)와 전체 자막 배열(CDN)이 결합되어 100% 동기화된 시청 경험을 제공합니다. 북마크 등 다른 액션 버튼이 배제되어 플레이어 제어와 자막 읽기라는 본질만 남습니다.

### 3.1. 검색 결과 네비게이션

- 결과 요약: 플레이어 상단(GNB 글로벌 검색창 바로 아래)에 "'{검색어}' 원어민 영상 (현재 순서 / 전체 매칭 건수)" 텍스트 노출 (text-secondary, type-caption).
- 영상 스와이프 컨트롤: [이전 영상], [다음 영상] 고스트 버튼을 큼직하게 양끝에 배치하여 숏폼 영상을 넘기듯 검색된 리스트를 빠르게 순회. 데스크탑 환경에서는 키보드 좌/우 방향키를 통한 즉각적인 영상 전환 지원.

### 3.2. 영상 플레이어 (YouTube IFrame)

- 뷰포트 렌더링: 16:9 비율 고정. 모바일에서는 화면 너비에 꽉 차게(Full-width) 배치하며 좌우 안전 여백 무시.
- 제어 권한 탈취: 유튜브 기본 자막 및 추천 영상 UI가 커스텀 자막 뷰어와 겹치지 않도록 IFrame API 파라미터(cc_load_policy=0, rel=0, controls=0)를 주입하여 기본 UI 강제 비활성화.
- 스마트 재생 시작점: 검색된 문맥을 자연스럽게 파악할 수 있도록, 대상 자막 청크의 절대 시작 시간(start_time)보다 0.5초에서 1초 앞선 시점부터 영상을 자동 재생.

### 3.3. 네이티브 청크 자막 뷰어 (Native Chunk Viewer)

자동 생성 자막(ASR)의 인지 부조화를 배제하고, 작성자의 호흡이 담긴 수동 자막 원본 청크를 그대로 렌더링하는 핵심 영역입니다.

- 레이아웃: 플레이어 바로 하단에 부착. 자막 텍스트 길이에 따른 상하 레이아웃 시프트(Layout Shift)를 방지하기 위해 컨테이너의 최소 높이(Min-height)를 강제 고정.
- 완전 동기화: CDN에서 비동기로 로드한 전체 자막 JSON 배열을 기반으로, requestAnimationFrame을 통해 60fps로 현재 영상 재생 시간에 맞춰 텍스트 전체 즉각 스왑(Swap).
- 모션 제로: 텍스트 교체 시 페이드인/아웃 등의 애니메이션 절대 금지(motion-duration-instant 적용).
- 텍스트 스타일: 시선 분산을 막고 주변 시야로 편하게 읽을 수 있도록 일반 본문보다 큰 type-subtitle-chunk 적용. 어절 단위 줄바꿈(word-break: keep-all) 유지.
- 정적 하이라이트 (Static Highlight): 화면에 노출 중인 청크 내에 사용자의 '검색어'가 포함된 경우, 해당 문자열에만 brand-highlight(노란색 배경) 및 text-inverse(검은색 텍스트) 강제 적용. 글자색이 변하는 노래방 효과 전면 금지.

### 3.4. 커스텀 어학 컨트롤러 (Player Controls)

자막 뷰어 하단 중앙에 배치되며, 어학 학습에 필수적인 제어 권한만을 직관적으로 제공하는 선형(Outline) 아이콘 조작부입니다. 터치 오작동 방지를 위해 요소당 최소 48px 이상의 투명 터치 영역을 강제합니다.

- 미세 탐색: [5초 뒤로], [5초 앞으로] 이동 버튼.
- 다시 듣기 (핵심 기능): 시청 흐름을 강제로 가두는 시스템 차원의 오토 루프(Auto-loop)를 배제하고 사용자에게 주도권 부여. 컨트롤러 정중앙에 1.2배 크기(Scale)로 강조하여 배치. 터치 시 현재 렌더링 중인 자막 청크의 원본 시작 시간(start_time)으로 즉시 점프. (단축키: R 또는 스페이스바).
- 재생 속도 조절: 글로벌 학습자 필수 기능. 0.75x, 1.0x(기본), 1.25x 순환 토글 텍스트 버튼.

## 4. 핵심 유저 플로우 (Core MVP User Journey)

MVP의 성패는 이 4단계의 학습 루프(Core Loop)가 얼마나 매끄럽고(Seamless) 빠르게(Superfast) 순환하느냐에 달려 있습니다. 어떠한 마찰(Friction)도 없이 제품의 '아하 모먼트'에 도달하는 최단 경로 시나리오입니다.

1. **탐색 (Discovery)**: 유저가 홈 화면에 접속하여 추천 칩을 터치하거나, 대형 검색창에 궁금한 한국어 표현을 입력하여 검색을 실행합니다.
2. **몰입 (Contextual Immersion)**: 화면이 플레이어 뷰로 전환되며, 매칭된 첫 번째 영상이 발화 시점 0.5초 전부터 자동 재생됩니다. 유저는 영상에 시선을 둔 채, 하단 자막 뷰어에서 노란색으로 정적 하이라이트된 검색어를 주변 시야로 확인하며 원어민의 억양과 상황을 파악합니다.
3. **통제 및 반복 (Control & Repeat)**: 해당 문맥 구간 시청이 끝나면, 유저는 컨트롤러 정중앙의 [다시 듣기] 버튼을 명시적으로 눌러 본인의 호흡에 맞춰 해당 문장을 원하는 만큼 반복 청취합니다. 충분히 이해했다면 [다음 영상] 버튼을 눌러 똑같은 표현이 다른 상황에서 어떻게 쓰이는지 숏폼을 보듯 넘기며 예시를 확인합니다.
4. **연속 탐색 (Continuous Search)**: 영상을 보던 중 파생된 궁금증이나 새로운 단어가 생각나면, 뒤로 가기를 누르지 않고 화면 상단(GNB)에 항상 떠 있는 글로벌 검색창에 새로운 단어를 즉시 입력하여 흐름 끊김 없이 1번 단계로 무한 루프를 이어갑니다.
